Daily data import

6 daily feed files. Trade file has individual trades — aggregated by UCC+segment on import. Brokerage is a separate file. Opening balance (not closing). Last trade date updated daily from trade file.

Import logic: (1) Trade file: individual trades → aggregate turnover by UCC+segment+date → update last_trade_date per UCC. (2) Brokerage file: stored in daily_trades alongside turnover. (3) Holdings: compute Σ(ISIN × Bhavcopy price × qty) per UCC → store only total in holdings_summary. ISIN detail discarded after compute.

Columns: UCC, Trade date, Segment (EQ_CASH / EQ_FUT / EQ_OPT / COMM_FUT / COMM_OPT), Turnover amount. Aggregated to daily_trades on import. Updates last_trade_date.

Nudge settings

Configure live order-placement nudges based on client trade history insights

What nudges are: When a client is about to place an order, the system checks their 90-day trade history and surfaces a personalised data observation on the order screen — e.g. "Your last 14 Far OTM NIFTY CE trades had a 38% win rate." These are statistical summaries of the client's own data, not investment advice. Client opt-in consent is required.
